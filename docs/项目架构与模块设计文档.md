## 3. 核心模块详细设计

### 3.1 SerialEngine（串口引擎）

**职责**：封装 QSerialPort，管理单个串口的连接、读取、写入、重连。

```
┌─────────────────────────────────┐
│         SerialEngine             │
├─────────────────────────────────┤
│ - port_: QString                │
│ - baudrate_: qint32             │
│ - databits_: QSerialPort::DataBits│
│ - parity_: QSerialPort::Parity  │
│ - stopbits_: QSerialPort::StopBits│
│ - serial_: QSerialPort*         │
│ - thread_: QThread*             │
│ - reconnect_timer_: QTimer*     │
├─────────────────────────────────┤
│ + open(port, baudrate, ...)     │
│ + close()                       │
│ + isOpen(): bool                │
│ + sendText(data: QString)       │
│ + sendHex(hex: QByteArray)      │
│ + setAutoReconnect(enabled)     │
├─────────────────────────────────┤
│ ◉ signal: dataReceived(QByteArray)│
│ ◉ signal: statusChanged(Status) │
│ ◉ signal: errorOccurred(QString)│
└─────────────────────────────────┘
```

**线程模型**：
- `SerialEngine` 实例运行在独立 `QThread` 中
- 通过 `moveToThread()` 将 QSerialPort 移到工作线程
- 使用信号槽与主线程通信（线程安全）

**重连策略**：
```
重连间隔: 1s → 2s → 3s → 5s → 5s → ...
成功连接后重置为 1s
```

### 3.2 LogBuffer（日志缓冲区）

**职责**：线程安全的固定大小环形缓冲区，存储解析后的日志条目。

```cpp
struct LogEntry {
    QString timestamp;      // "10:23:45.123"
    QString level;          // "INFO", "ERROR", "WARN", "DEBUG", "TRACE", ""
    QString text;           // 原始文本行
    QByteArray rawBytes;    // 原始字节（用于 HEX 显示和导出）
    QString portName;       // 来源串口名
};

class LogBuffer {
public:
    explicit LogBuffer(int maxSize = 10000);

    void append(const LogEntry& entry);
    void clear();
    QVector<LogEntry> getAll() const;
    QVector<LogEntry> getRecent(int count) const;
    QVector<LogEntry> getFiltered(const QString& keyword) const;
    int size() const;
    void setMaxSize(int size);

private:
    std::deque<LogEntry> buffer_;
    mutable QMutex mutex_;
    int maxSize_;
};
```

### 3.3 LogParser（日志解析器）

**职责**：解析原始字节流，识别日志级别，生成 LogEntry。

```cpp
class LogParser {
public:
    // 从原始字节提取文本行
    static QStringList extractLines(const QByteArray& data, QString& remainder);

    // 识别日志级别
    static QString detectLevel(const QString& line);

    // 生成带时间戳的显示文本
    static QString formatDisplay(const LogEntry& entry, bool showTimestamp);

    // 生成 HEX 显示文本
    static QString formatHex(const QByteArray& data, int offset);

    // 获取日志级别对应的颜色
    static QColor levelColor(const QString& level);
};
```

**日志级别识别规则**：
```
行中包含 "ERROR"   → ERROR
行中包含 "WARN"    → WARN
行中包含 "INFO"    → INFO
行中包含 "DEBUG"   → DEBUG
行中包含 "TRACE"   → TRACE
其他               → ""（普通文本）
```

### 3.4 ConfigManager（配置管理器）

**职责**：使用 rapidjson 读写 JSON 配置文件。

```cpp
struct SerialConfig {
    QString port;
    int baudrate = 115200;
    int databits = 8;
    QString parity = "N";
    int stopbits = 1;
    QString flowcontrol = "N";
};

struct DisplayConfig {
    QString theme = "dark";
    bool showTimestamp = true;
    bool hexMode = false;
    int fontSize = 12;
    int bufferSize = 10000;
    bool autoScroll = true;
};

struct AppConfig {
    SerialConfig serial;
    DisplayConfig display;
    QStringList filterHistory;
    QStringList sendHistory;
    QString sendAppend = "CRLF";
    int windowWidth = 1000;
    int windowHeight = 700;
    QVector<TabConfig> tabs;
};

class ConfigManager {
public:
    static ConfigManager& instance();

    bool load(const QString& path = "");
    bool save(const QString& path = "");
    AppConfig& config();
    QString configPath() const;

private:
    AppConfig config_;
    QString configPath_;
};
```

**配置文件路径优先级**：
1. 命令行 `--config` 指定路径
2. 环境变量 `SERIAL_MONITOR_CONFIG`
3. 默认路径：`%APPDATA%/EmberIntel/serial-monitor/config.json`

### 4.2 LogView（日志视图）

**核心设计：虚拟列表渲染**

为避免 10000 行日志导致 GUI 卡顿，LogView 采用虚拟列表模式：
- 仅渲染当前可见区域的日志行（约 50-100 行）
- 使用 `QAbstractScrollArea` 自定义绘制
- 滚动时动态计算可见行索引

```cpp
class LogView : public QAbstractScrollArea {
public:
    void setEntries(const QVector<LogEntry>& entries);
    void appendEntry(const LogEntry& entry);
    void clear();
    void setHexMode(bool enabled);
    void setShowTimestamp(bool enabled);
    void setAutoScroll(bool enabled);
    void setFilter(const QString& keyword);

protected:
    void paintEvent(QPaintEvent* event) override;
    void resizeEvent(QResizeEvent* event) override;

private:
    int visibleStartIndex() const;
    int visibleEndIndex() const;
    int lineHeight() const;
    void drawLine(QPainter& painter, int index, int y);
};
```

## 7. 线程安全设计

| 共享资源 | 保护机制 | 访问者 |
|----------|----------|--------|
| LogBuffer | QMutex | SerialEngine 写入，LogView/IPCServer 读取 |
| AppConfig | QMutex + 单例 | ConfigManager 读写，SettingsDialog 读写 |
| QSerialPort | 所属 QThread | 仅 SerialEngine 工作线程访问 |
| IPC 客户端列表 | QMutex | IPCServer 管理，主线程访问 |

## 8. 错误处理策略

| 错误类型 | 处理方式 |
|----------|----------|
| 串口打开失败 | 显示错误对话框，自动重试 |
| 串口运行时断开 | 自动重连，状态栏提示 |
| 配置文件损坏 | 恢复默认配置，日志警告 |
| IPC 连接断开 | CLI 提示重连，GUI 清理客户端 |
| 日志导出失败 | 显示错误提示，不中断程序 |
| 权限不足 | 自动申请管理员权限重启 |
